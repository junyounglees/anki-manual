탐색 창은 여러분의 카드와 노트들을 검색하고 수정할 수 있도록 해줍니다. 메인 화면에서 탐색을 클릭하거나 B를 눌러서 열 수 있습니다. 탐색 창은 왼쪽의 사이드바, 우측 상단의 카드/노트 테이블, 그리고 우측 하단의 수정 영역(editing area) 세 부분으로 구성됩니다. 두 영역 사이에 마우스를 위치시켜서, 드래그를 통해 한 영역의 크기를 키우거나 줄일 수 있습니다.

테이블 모드

Anki 2.1.45 이후 버전은 데이터 테이블에 카드들을 표시하거나 노트들을 표시하는 두 가지 모드를 제공합니다. 화면 상단의 검색 영역 왼쪽에 있는 스위치를 클릭하거나 Alt+T를 눌러서 현재의 모드를 변경할 수 있습니다. 이 스위치는 현재 Cards와 Notes 중 어떤 것이 보여지고 있는지 표시해줍니다.

참고: 간편함을 위해서, 이 메뉴얼은 일반적으로 Cards 모드가 켜진 것으로 가정합니다. 선택, 검색 등의 상황에서 "카드"가 언급된다면, 여러분들이 활성화한 모드에 따라서 카드 혹은 노트 모드로 변경해야할 것입니다.

사이드바

왼쪽의 사이드바 는 자주 사용하는 검색어에 빠르게 접근하도록 도와줍니다. Anki 2.1.45 이후 버전에서 사이드바는 또한 검색창을 제공하며, 태그와 뭉치를 수정할 수 있고, 아래에서 설명하게 될 2가지 도구를 선택할 수 있도록 해줍니다. 여러분은 사이드바 상단에 있는 툴바를 이용하거나 단축키 Alt+1/2를 통해 도구를 전환할 수 있습니다.

검색 도구

이 도구를 사용하면, 사이드바는 예전 버전에서처럼 동작합니다: 항목을 클릭하면 그것을 검색해줍니다.

여러분은 Ctrl (Mac에서는 Command)키를 누른채로 클릭하면, 해당 항목을 새롭게 검색하지 않고 현재의 검색에 AND 조건으로 덧붙일 수 있습니다. 예를 들어서 만약 여러분이 독일어 뭉치에 속해있는 익힘(learning) 카드를 표시하고 싶다면, "익힘 카드"를 클릭한 후 "독일어" 뭉치를 Ctrl-클릭할 수 있습니다.

AND 대신에 OR 검색을 하고 싶다면, Shift를 누를 수 있습니다. 예를 들어, 하나의 뭉치를 클릭한 다음 Shift-클릭으로 다른 뭉치를 클릭해서 두 뭉치 중 하나에 속하는 카드들을 한번에 표시할 수 있습니다.

여러분은 검색결과를 (- 기호를 앞에 붙여서) 부정문으로 바꾸기 위해서 Alt (Mac에서는 Option)를 누를 수 있습니다. 예를 들어, 현재의 뭉치에서 특정 태그를 포함하지 않는 모든 카드들을 보여주기 위해서 말이죠. Alt/Option 키는 Ctrl 또는 Shift와 조합해서 사용할 수 있습니다. 예를 들어, Ctrl+Alt를 누르면서 클릭하는 것은 부정문으로 바뀐 검색어를 (AND 조건으로) 추가합니다.

Anki 2.1.39 이후 버전에서, 여러분은 같은 방식의 검색결과를 새로운 결과로 바꾸기 위해서 Ctrl과 Shift키를 동시에 누른 상태로 검색어를 클릭할 수 있습니다. 여러분이 deck:Swahili (is:due or tag:important)^[코리안키 주) 이 검색어는 상태(is)가 복습카드(due)이거나 'important' 태그를 포함하고 있는 카드를 Swahili라는 뭉치(deck)에서 검색합니다.]과 같은 복잡한 검색어 표현을 이미 수행했다고 가정해보겠습니다. 이제 여러분의 Urdu 뭉치에서 똑같은 검색을 수행하고 싶습니다. 여러분은 Ctrl+Shift를 누른 상태에서 Urdu 뭉치를 클릭해서 다음과 같은 검색결과를 얻을 수 있습니다: deck:Urdu (is:due or tag:important).

선택 도구

선택 도구는 Ctrl 또는 Shift키를 누른 상태로 클릭해서 여러 개의 아이템을 선택할 수 있도록 해줍니다. 또한 뭉치와 태그를 재배열하기 위해서 드래그-앤-드랍을 가능하게 해줍니다.

예를 들어보겠습니다. 여러분이 Math, Calculus, 그리고 Algebra라는 태그를 갖고 있다고 해보겠습니다. Calculus 태그를 클릭하고, Ctrl-클릭으로 Algebra 태그를 클릭합니다. 이제 두 태그가 동시에 선택되었고, 두 태그를 클릭한 채로 Math 태그 위로 드래그해서 이 태그의 하위항목(children)으로 만들겠습니다. 내부적으로, Anki는 이 2개의 태그의 이름을 각각 Math::Calculus 와 Math::Algebra로 변경하고, 여러분의 노트들도 마찬가지로 업데이트합니다.

여러 항목을 선택하는 또다른 예시는 검색입니다: 여러 항목을 선택한 후 우클릭을 하면, 여러분은 Search > All/Any Selected를 고를 수 있습니다. 이것은 검색결과를 현재의 검색에 추가하기 위해 검색 도구에서 설명했던 키보드 조작법과 결합될 수 있습니다.

검색어 저장(Saved Searches)

만약 여러분이 동일한 것을 반복적으로 검색한다면, 사이드바에서 맨 위의 항목을 우클릭하고, "Save Current Search"를 선택하고, 이름을 입력해서 현재의 검색어를 저장할 수 있습니다. 여러분은 또한 사이드바의 어떠한 항목이든 이 영역으로 드래그 앤 드랍해서 저장된 검색어를 추가하고, 상단에 고정할 수 있습니다.

항목 수정하기

여러분은 사이드바에서 메뉴를 우클릭하거나 단축키(Del 그리고 Windows에서는 F2)를 사용해서 직접 태그, 뭉치, 저장된 검색어를 삭제하거나 이름을 변경할 수 있습니다. 삭제 기능은 여러 개의 항목에도 한번에 적용이 가능합니다. (선택 도구 참고).

항목 찾기

사이드바의 특정 항목을 찾기 위해서는, 상단에 있는 검색창에 이름의 일부분을 입력해서 일치하지 않는 모든 항목들을 일시적으로 숨길 수 있습니다.

검색 상자(Search Box)

카드 목록의 위에 있는 것은 검색 상자입니다. 여러분은 카드를 검색하기 위해서 여기에 다양한 것들을 입력할 수 있습니다. 검색 문법에 대한 정보는 Searching를 참고하세요.

카드/노트 테이블

테이블의 행(the table's row)은 현재의 검색과 일치하는 카드 또는 노트들을 보여줍니다. 하나의 행을 클릭하면, 해당되는 노트가 하단에 표시됩니다.

행(Rows)

여러 행을 선택하기 위해서 마우스를 드래그하거나 Ctrl 또는 Command키를 누르게 되면, 에디터(the editor)는 일시적으로 가려집니다. 활성화된 모드와 무관하게, 다양한 명령어들(뭉치 바꾸기 등)을 여러 개의 카드와 노트들에 한번에 수행할 수 있습니다. 따라서 카드 모드에서 특정 노트의 카드들 중 하나라도 선택되면 그 노트는 선택된 것으로 인식되며, 노트 모드에서 하나의 카드가 포함된 노트가 선택되면 그 카드는 선택된 것으로 인식됩니다.

(카드 정보 표시와 같은) 다른 명령어들은 오직 하나의 카드 또는 노트에 대해서만 실행됩니다. 이것은 현재 카드 또는 노트(current card or note)라고 부르는데, 일반적으로 가장 마지막에 선택되거나 클릭된 카드를 말합니다. 아까와 같이, 카드 모드에서 현재 노트(the current note)는 현재 카드의 노트(the note of the current card)이며, 노트 모드에서 현재 카드(the current card)는 현재 노트의 첫번째 카드(the first card of the current note)입니다.

카드와 노트에 따라 배경색깔이 바뀌게 됩니다. 카드 모드에서, 아래 순서에 따라 일치하는 첫번째 항목이 적용됩니다: d

  1. 카드에 플래그(표시)되었다면, 플래그의 색을 사용
  2. 카드가 일시중단(suspended)되었다면, 노란색
  3. 카드의 노트가 Mark되었다면, 보라색

노트 모드에서, 색깔은 오직 Mark된 노트에만 적용됩니다. Mark된 노트와 일시중단(suspended)된 카드에 대한 추가적인 정보는 편집하기 및 기타사항를 참고하세요.

열(Columns)

열(columns)은 설정이 가능합니다: 표시하고자 하는 열을 선택하기 위해서 하나의 열을 우클릭(또는 Mac에서는 Ctrl-클릭)하세요. 열을 드래그하면 재배열할 수 있습니다. 열을 클릭하면 해당 열에 따라 정렬됩니다; 정렬 순서를 뒤집으려면 다시 한번 클릭하세요. 질문과 답 열(Question and Answer columns)로는 정렬할 수 없다는 점을 참고하세요.

모든 열은 카드 모드와 노트 모드에서 모두 사용가능하지만, 약간 다른 이름과 데이터로 표시되는 경우도 있습니다. 아래 표는 두 모드들에 따라 표시하는 기능을 나열합니다.

편집 영역

The bottom right area displays the note of the currently selected row. For more information about cards and notes, see Getting Started. For more information on formatting buttons, see Editing.

You can see a preview of what the currently selected card would look like when reviewing by clicking the Preview button at the top of the editing area. Note that this will not display any type-the-answer fields on your cards, which makes it easier to preview the cards quickly. In Notes mode, the preview is shown for the first card of the selected note.

Cards

The following actions operate on the currently selected cards. They are also available through a context menu when a selected row is rightclicked in Cards mode. In Notes mode, they can be found in a submenu of the context menu.

NameAction
Change DeckMove currently selected cards to a different deck.
Set Due DateTurn cards into review cards, and make them due on a certain date. This can be useful for moving cards forward or back a few days when your study schedule is interrupted. Entering a range like 60-90 will make the selected cards due between 60 and 90 days from now. New cards will have their interval set to the same delay, but reviews will be rescheduled without changing their current interval, unless '!' is included at the end of the range.
ForgetMove currently selected cards to the end of the new queue. The existing review history is preserved. In 2.1.50+, there are options to restore the original card position, and to reset the card's lapse and repetition counters.
RepositionChange the order new cards will appear in. You can find out the existing positions by enabling the due column, as described in the table section above. If you run the reposition command when multiple cards are selected, it will apply increasing numbers to each card in turn. By default the number increases by one for each card, but this can be adjusted by changing the "step" setting. The Shift position of existing cards option allows you to insert cards between currently existing ones, pushing the currently existing ones apart. For instance, if you have five cards and you want to move 3, 4, and 5 between 1 and 2, selecting this setting would cause the cards to end up in the order 1, 3, 4, 5, 2. By contrast, if you turn this option off, 1 and 2 will get the same position number (and it will thus be unpredictable which of the cards with the same number comes up first). Please note that when enabled, any card with a higher position will be modified, and all of those changed cards will need to be sent the next time you sync.
Toggle SuspendSuspend or unsuspend all selected cards, depending on whether the current card is suspended or not.
FlagToggle the flags of all selected cards. Whether a flag is added or removed depends on whether the current card has the chosen flag.
InfoShow various information about the current card, including its review history. For more information, see Card Info.

Find and Replace

This dialog allows for replacing text on notes. As described above, it is available from the toolbar and the table's context menu.

The first input field is for the text that is going to be replaced, the second one for the replacement. Next, there is a dropdown menu that allows you to specify where Anki should look for text to replace: in a note's tags (requires Anki 2.1.45+), in all fields, or just in a specific field (only fields belonging to a selected note will be listed).

By default, only selected notes will be affected. If you want to lift that restriction, you can untick the "selected notes only" checkbox (requires Anki 2.1.45+).

Finding Duplicates

You can use the Notes > Find Duplicates option to search for notes that have the same content. When you open the window, Anki will look at all of your note types and present a list of all possible fields. If you want to look for duplicates in the Back field, you’d select it from the list and then click Search.

By default, it will search in all note types that have the field you provided. This differs from the duplicate check when you add cards manually, which is limited to a single note type.

The Optional filter text box allows you to narrow down where Anki will look for duplicates. If you only want to search for duplicates in the "French Vocab" and "French Verbs" note types, you would enter:

"note:french vocab" or "note:french verbs"

Or you might want to look only for duplicates in a particular deck, so you could use:

"deck:myDeck"

The search syntax is the same as used when searching in the browser. For more information, see Searching.

You can click one of the links in the search results list to display the duplicate notes in that set. If the search brings up a large number of duplicates, you may wish to instead click the Tag Duplicates button, which will tag all matching notes with duplicate. You can then search for this tag in the browser and handle them all from the same screen.
